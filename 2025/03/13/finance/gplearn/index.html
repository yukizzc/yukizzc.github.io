gplearn

Created2025-03-13|Updated2026-03-29|finance
|Post Views:

基本概念

gplearn 是一个基于**遗传编程（Genetic Programming, GP）**的 Python 库，用于自动生成符号表达式（symbolic expressions）。它受到经典遗传编程工具（如 GPLearn）的启发，特别适合从数据中挖掘复杂的数学关系，比如量化交易中的因子生成

遗传编程是遗传算法（Genetic Algorithm, GA）的一种变种，核心思想模仿自然进化：

  • 种群（Population）: 一组随机生成的候选解（在这里是数学表达式）
  • 适应度（Fitness）: 每个解的质量评分，通常是目标函数的误差（如 MSE）或收益指标
  • 进化操作
    • 选择（Selection）: 挑选适应度高的个体
    • 交叉（Crossover）: 将两个表达式的子树交换，生成新表达式
    • 变异（Mutation）: 随机改变表达式的部分结构（如替换操作符或变量）
  • 迭代进化: 通过多代进化，找到最优解

gplearn 的工作流程

  1. 初始化种群: 随机生成大量数学表达式（如 X0 + X1, sin(X2) 等）
  2. 评估适应度: 对每个表达式计算目标值（如预测误差）
  3. 进化迭代: 通过选择、交叉和变异生成新一代种群
  4. 终止条件: 达到最大代数或适应度满足阈值后停止
  5. 输出最佳表达式: 返回适应度最高的符号表达式

基本设置

gplearn 是一个基于遗传编程（Genetic Programming, GP）的 Python 库，提供了三个核心类，用于不同的任务场景。它们都利用遗传编程的进化机制（选择、交叉、变异）从数据中生成符号表达式，但目标和应用方向各有侧重

SymbolicRegressor

概述

  • 功能: 用于回归任务，通过遗传编程生成一个符号表达式来预测连续的目标值
  • 适用场景: 在量化交易中生成因子表达式，预测股票收益率、价格变化等连续变量
  • 输出: 一个最优的数学表达式（如 add(X0, mul(X1, 2))），直接用于预测

核心参数

  1. population_size: 整数，默认 1000
    • 定义每代种群中候选表达式的数量。
    • 越大搜索范围越广，但计算成本增加
  2. generations: 整数，默认 20
    • 进化迭代的次数
    • 代数越多，可能找到更优解，但也可能过拟合
  3. function_set: 元组或列表，默认 (‘add’, ‘sub’, ‘mul’, ‘div’)
    • 指定表达式中可用的数学函数，例如 ‘sqrt’、‘log’、‘sin’
    • 支持自定义函数
  4. metric: 字符串或自定义函数，默认 ‘mse’
    • 适应度评估指标：
      • ‘mse’: 均方误差
      • ‘rmse’: 均方根误差
      • ‘mae’: 平均绝对误差
      • 可自定义（如夏普比率）
  5. tournament_size: 整数，默认 20
    • 锦标赛选择时每次比较的个体数
    • 值越大，选择压力越大，收敛更快
  6. stopping_criteria: 浮点数，默认 0.0
    • 当最佳适应度小于此值时停止进化
    • 例如设为 0.01，表示 MSE < 0.01 时结束
  7. const_range: 元组，默认 (-1.0, 1.0)
    • 表达式中随机常数的取值范围
    • 可设为 (0, 100) 以生成更大常数
  8. init_depth: 元组，默认 (2, 6)
    • 初始表达式树的深度范围
    • 控制初始种群的复杂度
  9. p_crossover: 浮点数，默认 0.9
    • 子代通过交叉操作生成的概率
    • 值高则更倾向于组合现有表达式
  10. p_subtree_mutation: 浮点数，默认 0.01
    • 子树变异的概率，随机替换表达式的一个子树
  11. parsimony_coefficient: 浮点数，默认 0.0
    • 惩罚复杂表达式的系数，值越大越倾向于简单表达式
  12. verbose: 整数，默认 0
    • 是否显示进化过程，设为 1 显示每代统计
  13. random_state: 整数或 None，默认 None
    • 随机种子，确保结果可复现

补充说明

  • SymbolicRegressor 假设输入 X 是特征矩阵，y 是连续目标值
  • 训练后，gp._program 属性返回最佳表达式

SymbolicTransformer

概述

  • 功能: 用于特征生成，从输入数据中自动创建多个新的符号表达式作为特征，而不是直接预测目标
  • 适用场景: 在量化中生成一组潜在因子，供后续模型（如线性回归、机器学习）使用
  • 输出: 一组最佳表达式（数量由 n_components 决定），转化为新的特征矩阵

核心参数

  1. n_components: 整数，默认 10
    • 指定生成的特征（表达式）数量
    • 例如设为 5，返回 5 个最佳特征
  2. population_size: 整数，默认 1000
    • 每代种群中候选表达式的数量
    • 与回归类相同，但影响特征多样性
  3. generations: 整数，默认 20
    • 进化代数，决定特征生成的迭代次数
  4. function_set: 元组或列表，默认 (‘add’, ‘sub’, ‘mul’, ‘div’)
    • 可用的数学操作符，支持扩展（如 ‘log’、‘exp’）
  5. hall_of_fame: 整数，默认 100
    • 保存的最佳个体数量，从中挑选 n_components 个特征。
    • 值越大，保留的优秀表达式越多。
  6. tournament_size: 整数，默认 20
    • 锦标赛选择的大小，影响进化速度
  7. metric: 字符串或自定义函数，默认 ‘pearson’
    • 特征选择的适应度指标：
      • ‘pearson’: 皮尔逊相关系数（与目标的相关性）
      • ‘spearman’: 斯皮尔曼相关系数
      • 可自定义
  8. p_crossover: 浮点数，默认 0.9
    • 交叉操作的概率，生成新特征的倾向性
  9. p_point_mutation: 浮点数，默认 0.01
    • 点变异的概率，随机替换单个节点
  10. init_method: 字符串，默认 ‘half and half’
    • 初始种群生成方式：
      • ‘grow’: 随机生长
      • ‘full’: 满树生成
      • ‘half and half’: 混合模式
  11. verbose: 整数，默认 0
    • 是否打印进化日志
  12. random_state: 整数或 None，默认 None
    • 控制随机性

补充说明

  • SymbolicTransformer 不需要目标 y（除非自定义 metric），主要基于特征间的关系生成表达式
  • 使用 fit_transform(X) 返回新特征矩阵

SymbolicClassifier

概述

  • 功能: 用于分类任务，生成符号表达式预测离散类别（如上涨/下跌）
  • 适用场景: 在量化中预测股票价格方向（二分类）或多分类问题
  • 输出: 一个分类表达式，返回概率或类别标签

核心参数

  1. population_size: 整数，默认 1000
    • 种群中候选表达式的数量
  2. generations: 整数，默认 20
    • 进化迭代次数
  3. function_set: 元组或列表，默认 (‘add’, ‘sub’, ‘mul’, ‘div’)
    • 可用的数学函数集，可加入 ‘max’、‘min’ 等
  4. metric: 字符串，默认 ‘log loss’
    • 分类任务的适应度指标：
      • ‘log loss’: 对数损失，优化概率预测
      • ‘accuracy’: 分类准确率
      • 可自定义（如 F1 分数）
  5. tournament_size: 整数，默认 20
    • 锦标赛选择的大小
  6. stopping_criteria: 浮点数，默认 0.0
    • 当适应度小于此值时停止
  7. class_weight: 字典或 None，默认 None
    • 类别权重，例如 {0: 1, 1: 2}，调整类别不平衡
  8. threshold: 浮点数，默认 0.5
    • 分类概率的阈值，用于将输出转为类别
  9. p_crossover: 浮点数，默认 0.9
    • 交叉概率
  10. p_hoist_mutation: 浮点数，默认 0.01
    • 提升变异概率，随机提取子树以简化表达式
  11. max_samples: 浮点数，默认 1.0
    • 用于计算适应度的样本比例，降低过拟合风险
  12. verbose: 整数，默认 0
    • 是否显示训练过程
  13. random_state: 整数或 None，默认 None
    • 随机种子

补充说明

  • SymbolicClassifier 假设 y 是离散标签（如 0 和 1）
  • 输出概率通过 predict_proba 获取，类别通过 predict 获取

总结与对比

类名 目标 输出 示例场景
SymbolicRegressor 预测连续值 单一表达式 预测收益率
SymbolicTransformer 生成新特征 多个表达式（特征矩阵） 生成因子组合
SymbolicClassifier 预测分类标签 单一表达式（概率/类别） 预测涨跌方向

进阶

  • 在 gplearn 中，自定义函数允许扩展默认的 function_set，加入特定的数学或金融操作。例如，默认的 function_set 包括 ‘add’, ‘sub’, ‘mul’, ‘div’ 等基本运算，但如果你想加入金融领域常用的指标（如 RSI、指数平滑等），可以通过 make_function 定义并添加到模型中
  • 在 gplearn 中，默认的 metric 是 ‘mse’（均方误差），但你可以通过 make_fitness 定义夏普比率作为适应度函数，替换默认指标

分析

训练中打印的信息如下

Gen: 进化代数，第 0 代是初始种群，第 9 代是最后一代

Population Average:

  • Length: 种群中表达式的平均长度（节点数）
    • 第 0 代：10.5
    • 第 9 代：8.2（变短，可能受 parsimony_coefficient 影响）
  • Fitness: 种群的平均适应度。
    • 第 0 代：1.234e+40（极大值，表明初始种群有很多无效表达式）
    • 第 9 代：0.00012（显著降低，适应度改善）

Best Individual:

  • Length: 最佳表达式的长度
    • 第 0 代：5
    • 第 9 代：6（稍复杂）
  • Fitness: 最佳表达式的适应度。
    • 第 0 代：0.000045
    • 第 9 代：0.000032（更低，模型优化成功）

OOB Fitness: 袋外适应度，未启用（N/A）

Time Left: 剩余时间，从 15.2 秒到 0.0 秒，表示训练完成

Length：的表达式长度指的就是公式复杂度，如下面这样的公式长度就是5，长度越长就表示公式越复杂也越容易过拟合，可通过parsimony_coefficient进行控制

Fitness：是衡量因子表达式性能的指标，具体取决于设置的 metric 参数
