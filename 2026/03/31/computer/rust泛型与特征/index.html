rust泛型与特征

Created2026-03-31|Updated2026-04-02|computer
|Post Views:

泛型

泛型允许定义可以处理多种类型的代码，避免重复编写相似的逻辑。泛型是 Rust 中实现代码复用和类型安全的核心机制。

核心优势：

  • 消除重复代码（不写泛型就要为每种类型写一个函数）
  • 编译时单态化，零运行时开销
  • 保持类型安全，编译器会检查类型约束

函数中的泛型

在函数签名中使用泛型，可以创建适用于多种类型的函数。通常需要为泛型添加trait 约束，限制可接受的类型。

语法说明：

  • T 是泛型参数（通常使用大写字母：T、U、V）
  • :PartialOrd 是 trait 约束，表示 T 必须实现比较功能
  • 调用时编译器会根据传入参数自动推断具体类型

结构体中的泛型

结构体可以使用泛型定义字段类型，使结构体更灵活。

使用场景：

  • 创建通用的数据结构（如 Vec<T>、HashMap<K, V>）
  • 封装不同类型的数据
  • 实现类型安全的容器

方法中的泛型

为泛型结构体实现方法时，需要在 impl 后声明泛型参数。

关键点：

  • impl<T> 中的 T 是结构体的泛型参数
  • 方法中可以使用新的泛型参数（如 mixup<U> 中的 U）
  • 可以为特定类型（如 Point<f32>）单独实现方法

单态化

Rust 的泛型是零成本抽象：编译器在编译时将泛型代码转换为具体类型的代码，这个过程称为单态化

性能特点：

  • 泛型代码和手写具体类型代码性能完全相同
  • 没有运行时开销（不像 Java 的类型擦除）
  • 代价是编译时间增加和二进制体积变大（因为生成多份代码）

泛型约束

泛型约束用于限制泛型类型必须实现某些 trait，确保泛型代码可以调用特定方法。Rust 提供了三种写法：

写法 语法 适用场景
内联约束 fn func<T: Trait>(x: T) 约束少、简洁
where 子句 fn func<T>(x: T) where T: Trait 约束多、可读性好
impl Trait fn func(x: impl Trait) 参数简单、返回值位置

常量泛型

常量泛型允许将常量值（如数组长度）作为泛型参数，使代码可以处理不同大小的数组、矩阵等数据结构。

泛型总结

概念 说明
泛型参数 T、U 等，表示抽象类型
trait 约束 T: PartialOrd，限制可接受的类型
单态化 编译时生成具体类型代码，零运行时开销
常量泛型 const N: usize，将常量作为泛型参数

特征（Traits）

特征定义共享的行为，类似于其他语言的接口（interface）。特征允许不同类型实现相同的方法，实现多态。

特征基础

定义

使用 trait 关键字定义特征，可以包含方法签名和默认实现。

实现

使用 impl Trait for Type 语法为具体类型实现特征。

使用

特征作为参数

特征可以作为函数参数，接受任何实现了该特征的类型。Rust 提供三种写法：

写法 语法 适用场景
impl Trait fn f(item: &impl Trait) 简洁，单个参数
特征约束 fn f<T: Trait>(item: &T) 需要指定类型名
where 子句 fn f<T>(item: &T) where T: Trait 多约束、可读性好

单参数

多参数

特征作为返回值

使用 impl Trait 可以返回实现了某特征的类型，但只能返回单一具体类型。

基本用法

限制: 不能返回不同类型

特征对象（动态分发）

特征对象允许在运行时动态调用方法，通过 dyn Trait 语法，通常包装在 Box、& 或 Rc 中。

静态分发 vs 动态分发

特性 静态分发（泛型） 动态分发（特征对象）
实现方式 单态化，生成多份代码 vtable 表，一份代码
性能 更快（零开销） 稍慢（间接调用）
代码体积 较大（重复生成） 较小
灵活性 编译时确定 运行时确定
使用场景 性能敏感、类型已知 需要异构集合
