rust所有权与生命周期

Created2026-03-31|Updated2026-03-31|computer
|Post Views:

所有权核心规则

  1. Rust 中的每个值都有一个所有者（owner）
  2. 同一时刻只能有一个所有者
  3. 当所有者离开作用域，值会被自动释放（调用 drop）

作用域规则

移动

移动是 Rust 所有权的核心规则之一：当一个值被赋值给另一个变量时，该值的所有权会从源变量转移到目标变量，源变量随即失效，无法再被使用。

关键点：

  • Rust 默认通过移动来转移堆上数据的所有权，避免重复释放
  • 实现了 Copy trait 的简单类型（如整数、布尔值）会自动复制而非移动
  • 移动发生后，原变量被"废弃"，编译器会阻止对其的访问

克隆

克隆是对数据进行深拷贝的操作，复制后的数据拥有独立的内存空间，与原数据互不影响。

关键点：

  • clone() 方法会完整复制堆上的数据，产生一份全新的副本
  • 克隆后，原变量和新变量都有效，各自拥有独立的所有权
  • clone() 通常开销较大（尤其是 String、Vec 等容器类型）
  • 实现了 Copy trait 的类型（如整数）赋值时自动复制，无需手动调用 clone()

Copy trait 类型

Copy trait 标记类型在赋值或传递时会自动复制而非移动。实现 Copy 的类型通常存储在栈上，复制开销极低。

关键点：

  • 赋值或传参时自动复制，原变量仍然可用
  • 只能由简单类型（所有字段都实现 Copy）实现
  • 实现 Copy 必须先实现 Clone
  • 常见类型：整数、浮点数、布尔值、字符、引用、元组（字段均实现 Copy）

引用与借用

引用基础

引用是指向数据而不获取所有权的指针，允许在不转移所有权的情况下访问数据。

不可变引用（&T）：

  • 允许多个同时存在
  • 只能读取数据，不能修改
  • 数据在引用期间不能被修改

可变引用（&mut T）：

  • 同一时刻只能有一个
  • 可以修改数据
  • 存在期间不能有其他任何引用（不可变或可变）

悬垂引用

悬垂引用指向已被释放的内存，Rust 编译器会在编译时阻止这种情况

切片

切片是没有所有权的引用，指向集合中一段连续的内存区域。切片允许在不拷贝数据的情况下，只访问集合的一部分。

核心特点：

  • 切片是一种引用，不拥有数据所有权
  • 指向集合的连续部分，可以灵活截取
  • 切片本身是一个"胖指针"，包含数据指针和长度信息

字符串切片

字符串切片（&str）指向 String 或字符串字面量的部分内容。截取语法使用范围 [start..end]，end 是不包含的结束位置。

范围简写规则：

  • [0..n] → [..n]（省略开头）
  • [n..len] → [n..]（省略结尾）
  • [0..len] → [..]（省略两端）

数组切片

数组切片与字符串切片类似，可以截取数组的连续部分。

切片作为参数

使用 &str 作为参数类型可以同时接受 String 和字符串字面量，是 Rust 中的推荐写法。

&[T] 作为参数时可以接受数组引用 &[T; N] 和 Vector 引用 &Vec<T>，它们会自动解引用为切片，这与 &str 接受 &String 的机制完全相同，都是 Rust 的 Deref 强制转换特性

总结

参数类型 可接受的值
&str &String、字符串字面量 &str、String 的引用
&[T] &Vec<T>、&[T; N]、&[T] 切片

生命周期

生命周期是引用保持有效的作用域，Rust 通过生命周期检查器在编译时确保所有引用都是有效的，防止悬垂引用。

核心目的：保证引用始终指向有效的数据，避免内存安全问题。

生命周期标注语法

生命周期参数以撇号（'）开头，通常使用小写字母命名（如 'a、'b）。标注本身不改变引用的生命周期，只是建立引用之间的关系。

关键理解：标注是告诉编译器这些引用的生命周期存在某种关系，而不是手动指定存活时间。

函数中的生命周期

当函数返回引用且该引用来源于参数时，需要标注生命周期关系。

何时需要标注：

  • 返回值是引用
  • 返回值来源于多个输入引用之一
  • 编译器无法自动推断生命周期关系

结构体中的生命周期

当结构体包含引用时，必须为每个引用标注生命周期，确保结构体实例不会持有悬垂引用。

生命周期省略规则

编译器会自动推断生命周期的三条规则，满足时无需手动标注：

规则 说明 示例（省略写法） 示例（完整写法）
规则1 每个引用参数都有自己的生命周期 fn f(x: &i32, y: &i32) fn f<'a, 'b>(x: &'a i32, y: &'b i32)
规则2 只有一个输入引用时，输出生命周期与之相同 fn f(x: &i32) -> &i32 fn f<'a>(x: &'a i32) -> &'a i32
规则3 方法中，&self 的生命周期赋给所有输出引用 fn f(&self, x: &i32) -> &i32 fn f<'a, 'b>(&'a self, x: &'b i32) -> &'a i32

静态生命周期

'static 是特殊的生命周期，表示引用在整个程序运行期间都有效。

常见的 'static 使用场景：

  • 字符串字面量
  • 全局静态变量
  • 作为 trait 对象中的生命周期约束（如 Box<dyn Trait + 'static>）

生命周期总结

概念 要点
本质 引用有效的作用域，编译时检查
标注 'a，用于建立引用间的关系
省略规则 三条规则，减少手动标注
结构体 包含引用时必须标注
'static 整个程序运行期间有效
