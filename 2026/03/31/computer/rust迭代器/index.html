rust迭代器

Created2026-03-31|Updated2026-03-31|computer
|Post Views:

迭代器是 Rust 中处理元素序列的核心抽象，实现了零成本抽象和函数式编程风格。迭代器模式允许你按顺序访问集合中的每个元素，而不需要关心底层数据结构。Rust 的迭代器是惰性的，只有在需要结果时才会执行计算。

迭代器基础

迭代器是实现了 Iterator trait 的类型，其核心是 next() 方法，每次返回一个 Option<Self::Item>，当迭代结束时返回 None。

关键理解：

  • 迭代器是内部可变的，调用 next() 会改变迭代器的内部状态
  • Iterator trait 有许多默认实现的方法，只要实现 next()，就能免费获得大量功能（如 map、filter、sum 等）

三种迭代器类型

Rust 中的集合通常提供三种方式获取迭代器，区别在于元素的访问权限和所有权：

迭代器类型 获取方式 所有权 元素类型 适用场景
iter() 不可变引用 借用 &T 只读访问
iter_mut() 可变引用 借用 &mut T 需要修改元素
into_iter() 获取所有权 转移 T 消耗集合，转移所有权

使用建议：

  • 只需要读取数据 → 使用 iter()
  • 需要修改数据 → 使用 iter_mut()
  • 需要转移所有权（如将集合移动到新函数）→ 使用 into_iter()

迭代器适配器

适配器是返回新迭代器的方法，用于转换、过滤、组合迭代器。适配器是惰性的，它们不会立即执行，只有在调用消费者方法时才真正计算。

map转换元素

map 对每个元素应用一个函数，产生新的元素。这是最常用的适配器之一。

filter过滤元素

filter 只保留满足条件的元素，丢弃不满足的。
