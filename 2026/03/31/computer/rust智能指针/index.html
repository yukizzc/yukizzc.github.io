rust智能指针

Created2026-03-31|Updated2026-03-31|computer
|Post Views:

智能指针是行为类似指针但拥有额外元数据和功能的数据结构。Rust 中普通引用（&T）只借用数据，而智能指针通常拥有数据的所有权。

Box

Box<T> 是最简单的智能指针，将数据分配在堆上，栈上只保存指向堆数据的指针。

基本用法

适用场景

场景1：递归类型（最常见）

Rust 需要在编译时知道类型大小，递归类型无法确定大小，需要用 Box 间接存储。

场景2：大对象转移所有权

将大对象放在堆上，转移时只复制指针（8字节），避免复制大量数据。

场景3：特征对象

Box<dyn Trait> 用于存储实现了某特征的不同类型。

Deref 与 Drop

智能指针之所以"智能"，是因为它们实现了两个关键特征：

特征 作用 类比
Deref 允许像普通引用一样使用 * 解引用 让智能指针"像"普通指针
Drop 值离开作用域时自动执行清理代码 析构函数（RAII）

Deref：解引用特征

Deref 特征允许重载解引用运算符 *，使智能指针可以像普通引用一样使用。

Deref 强制转换：当类型实现了 Deref，Rust 会自动将引用转换为目标类型的引用。

转换规则：

  • &MyBox<T> → &T（通过 Deref::deref）
  • &mut MyBox<T> → &mut T（通过 DerefMut）

Drop：析构特征

Drop 特征在值离开作用域时自动执行清理代码，实现 RAII（资源获取即初始化）模式。

提前释放：使用 std::mem::drop 可以提前释放值。

Drop 与 Deref 的关系：

  • Deref 控制如何"访问"智能指针内部的数据
  • Drop 控制智能指针"离开时"如何清理

Rc

Rc<T>（Reference Counted）允许多个所有者共享同一份数据，通过引用计数管理生命周期。

重要：Rc<T> 仅适用于单线程环境，多线程使用 Arc<T>。

RefCell

RefCell<T> 提供内部可变性：即使有不可变引用，也能修改内部值。它将借用检查从编译时推迟到运行时。

核心概念

  • 目的：提供内部可变性
  • 关键：将借用检查从编译时推迟到运行时
  • 规则：模拟普通引用的借用规则（多个不可变/独占可变/互斥）
  • 代价：违反规则时 panic（运行时错误而非编译错误）
  • 使用场景：需要可变性但编译器无法证明安全时（如 Rc 配合使用）
特性 普通引用 &T RefCell<T>
检查时机 编译时 运行时
违反后果 编译错误 panic!
性能 零开销 运行时计数开销
灵活性 受限 更灵活

普通引用通过编译器的非词法生命周期（NLL），在最后一次使用后即视为借用结束，无需等到作用域结束；RefCell 的借用基于运行时计数器，必须等到变量 drop（离开作用域或显式调用 drop）才能释放。

结合 Rc 实现

Rc<RefCell<T>> 是常用组合，实现多所有权 + 内部可变性。

Cell

Cell<T> 是更简单的内部可变性实现，通过复制值而非借用来修改。

基本用法

限制

Cell<T> 要求 T 实现 Copy trait，适用于简单类型。复杂类型使用 RefCell。

引用循环与弱引用

引用循环问题

Rc<T> 可能导致引用循环，造成内存泄漏。

Weak弱引用

Weak<T> 是 Rc<T> 的弱引用版本，不增加强引用计数，可以打破引用循环。

总结

智能指针 多所有权 可变性 线程安全 主要用途
Box<T> ❌ 单一 可变（需 &mut） ✅ 可发送 堆分配、递归类型
Rc<T> ✅ 多个 不可变 ❌ 不 单线程共享只读
RefCell<T> ❌ 单一 运行时可变 ❌ 不 内部可变性
Rc<RefCell<T>> ✅ 多个 运行时可变 ❌ 不 单线程共享可变
Arc<T> ✅ 多个 不可变 ✅ 可 多线程共享只读
Mutex<T> ❌ 单一 运行时可变 ✅ 可 多线程共享可变
Weak<T> ✅ 多个 不可变 ❌ 不 打破引用循环
