函数

函数定义

Rust 支持函数指针类型（fn），可以将函数作为参数传递给其他函数。函数指针是一个具体类型，指向编译时已知的函数

高阶函数

高阶函数是指满足以下至少一个条件的函数：

  1. 以函数作为参数
  2. 返回一个函数

Rust 中高阶函数的实现方式主要有：

  • 函数指针（fn）
  • 闭包（Fn、FnMut、FnOnce traits）

作为参数

作为返回值

函数指针只能引用编译时确定的函数，无法捕获外部环境的变量；而闭包可以捕获其定义时所在作用域中的变量

闭包

闭包是可以捕获环境变量的匿名函数

基础

Fn、FnMut、FnOnce

Fn、FnMut、FnOnce 的区别

Trait 特点 捕获方式 调用次数
FnOnce 只能调用一次 移动捕获 1次
FnMut 可以修改捕获的变量 可变引用捕获 多次
Fn 不修改捕获的变量 不可变引用捕获 多次

自动推导规则

编译器会根据闭包体对捕获变量的操作方式，自动推断闭包实现了哪个 trait。你无需手动指定，Rust 会智能选择最合适的类型。

特殊场景

只有在以下情况需要留意(这三种最常见)：

  1. 将闭包作为函数参数时，需要选择合适的 trait 约束
  2. 多线程编程时，可能需要 move 关键字
  3. 返回闭包时，需要确保捕获变量的生命周期

闭包作参数

下面例子中如果把函数参数约束条件改成 Fn 就会报错，因为：

  1. 函数要求：process_data 的参数约束是 Fn，意味着传入的闭包不能修改环境变量
  2. 闭包实际：accumulate 内部执行了 sum += x，修改了外部变量
  3. 编译器推导：根据闭包行为，编译器自动将其推导为 FnMut（可修改环境）
  4. 类型不匹配：FnMut 无法传递给要求 Fn 的参数，因为 Fn 比 FnMut 更严格

这里F其实就是一个泛型约束，表面了哪一类闭包可以传入

多线程编程

多线程场景下需要使用 move 关键字，强制将捕获变量的所有权转移到闭包中，从而解决生命周期问题。

move 只影响变量所有权的转移方式，不影响闭包实现的 trait 类型。闭包类型仍然由闭包体决定：

闭包体行为 闭包类型 多线程示例
只读取变量 Fn `move
修改变量 FnMut `move
消耗变量 FnOnce `move

关键点：move 不是让闭包变成 FnOnce，只有闭包体消耗了捕获的变量才会成为 FnOnce

返回闭包

返回闭包时，需要特别注意捕获变量的生命周期问题。因为闭包会捕获环境中的变量，如果直接返回，这些变量可能会在函数结束后被销毁，导致悬垂引用
