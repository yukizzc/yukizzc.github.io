可变性

Rust 的变量默认是不可变的，需要修改时必须显式使用 mut 关键字声明。此外，Rust 支持变量遮蔽（shadowing），允许重新定义同名变量，甚至可以改变其类型。

标量数据

  • 整数：i8 到 i128, u8 到 u128, isize, usize
  • 浮点：f32, f64
  • 布尔：bool
  • 字符：char（注意是 Unicode 标量值，4 字节）

整数

rust中申明变量都需要确定类型，类型标识可以以下三种方式

  1. 编译器根据上下文自动推导，此时自己可以省略不写
  2. 写在变量名后面
  3. 写在赋值的数值后面，可以用下划线分割

整形溢出，对于运行时的变量计算rust在不同模式下会与不同行为，对于下面测试代码debug和release下的行为如表格所示

字符

Rust 的 char 类型表示 Unicode 标量值，使用 UTF-32 编码固定存储为 4 字节，因此支持全球所有 Unicode 字符（包括中文、emoji 等）

as转换

as 主要用于：

  1. 数值类型之间的转换（整数 ↔ 整数，整数 ↔ 浮点）
  2. char ↔ 数值（但 char → 数值安全，数值 → char 需要 from_u32）
  3. 指针/引用与 usize 之间的转换（unsafe 场景）
  4. 枚举转整数（需要 #[repr] 注解）

复合类型

元组

元组是定长、异构的数据集合，一旦定义长度不可变

元组在函数返回值中非常有用，可以返回多个数据

数组

数组是定长、同构的数据集合。

集合类型

Rust 标准库提供了多种集合类型，用于存储和管理数据。与数组和元组不同，集合类型通常在堆上分配，可以动态增长。

三种主要集合类型

String 字符串 UTF-8 编码的字符串 处理文本数据
HashMap<K, V> 哈希映射 键值对存储 通过键快速查找值

其他集合

VecDeque

VecDeque（双端队列）是一种支持两端高效操作的线性容器，可在队列头部和尾部进行 O(1) 时间的插入与删除，适用于需要频繁从两端添加或移除元素的场景

HashSet

HashSet 是基于哈希表实现的无序集合，存储唯一的键值（无重复元素），支持 O(1) 时间的插入、删除和查找，适用于需要去重、快速判断元素存在的场景

Option和Result

Option<T> 和 Result<T, E> 是 Rust 的类型安全错误处理机制，分别用于表示可能存在空值和可能失败的运算，强制调用者显式处理所有情况，避免空指针和未捕获异常

字符串 (String和&str)

Rust 的字符串处理是其核心特色之一，主要有两种类型：String 和 &str。理解它们的区别和使用场景至关重要

两种字符串类型对比

特性 String &str (字符串切片)
内存位置 堆 (heap) 栈/堆/静态区 (取决于来源)
可变性 可变 (mut 可修改) 不可变
所有权 拥有所有权 借用 (无所有权)
大小 运行时动态大小 编译时可知的指针大小 (16字节)
使用场景 需要修改、拼接、返回 只读操作、函数参数
