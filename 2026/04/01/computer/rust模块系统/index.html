rust模块系统

Created2026-04-01|Updated2026-04-01|computer
|Post Views:

模块系统是 Rust 组织代码的核心机制，包括 Package（包）、Crate（箱）、Module（模块）、Path（路径）、可见性控制等。良好的模块化设计能让代码更清晰、更易维护。

一个完整项目结构如下

Package 与 Crate

基本概念

概念 说明 类比
Crate 编译单元，编译后生成一个二进制文件或库文件 一个"项目"
Package 包含一个或多个 Crate，由 Cargo.toml 定义 一个"仓库"
Crate Root 编译器开始编译的入口文件 src/main.rs 或 src/lib.rs

关系：

  • 一个 Package 包含至少一个 Crate（通常是 1 个库 + 多个二进制）
  • Package 可以有最多一个库 crate，可以有任意多个二进制 crate

Package 结构

Crate 类型

二进制 crate 与库 crate 的区别：

  • 二进制 crate：有 main() 函数，可独立运行
  • 库 crate：无 main()，提供功能供其他项目使用

Module 与 Path

Rust 中，模块是组织代码的单元，路径则用于定位模块或其中的项。根据模块定义方式的不同，可以分为三种形式：

形式 适用场景 文件结构
内联模块 小型、临时代码块 直接写在 .rs 文件中
文件模块 单一功能模块 src/module.rs
目录模块 复杂功能（含子模块） src/module.rs + src/module/*.rs

内联模块（Inline Module）

在同一文件中使用 mod 关键字直接定义模块，模块内容写在花括号内。

文件模块（File Module）

将模块内容放在单独的文件中，文件名即为模块名。

目录结构：

src/utils.rs

src/main.rs

目录模块（Directory Module）

当一个模块较复杂时，可以拆分为一个目录，使用 同名文件 + 同名文件夹 的方式组织

目录结构：

src/utils.rs

src/utils/helper.rs

src/main.rs

路径系统（Path System）

Rust 中访问模块内的项需要使用路径，分为两种：

  • 绝对路径：从 crate 根开始，以 crate:: 开头
  • 相对路径：从当前模块开始，使用 self、super 或直接写

路径访问控制

  • pub：将模块或项暴露为公开，外部可访问
  • pub(crate)：仅当前 crate 内可见
  • pub(super)：仅父模块可见
  • pub(in path)：仅在指定路径内可见

use 关键字

当模块路径较长时，可以使用 use 关键字将路径绑定到当前作用域，简化重复调用。

可见性

Rust 默认所有项私有，需显式标记 pub 开放。

可见性规则表

项目 默认状态 公开方式 特殊说明
模块 私有 pub mod 模块公开后，内部项仍需各自标记 pub
结构体/枚举 私有 pub struct / pub enum 结构体字段默认仍私有
结构体字段 私有 pub field: Type 需逐个字段标记
Trait 私有 pub trait 公开后，实现该 trait 的方法自动公开
函数/方法 私有 pub fn Trait 实现的方法除外

核心原则：pub 只影响当前层级，子模块、字段、嵌套项需要各自的 pub。
