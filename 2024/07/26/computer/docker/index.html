镜像原理与制作

镜像分层原理

Docker 镜像由多个只读的镜像层（Layer） 堆叠而成，每个层记录了文件系统的变化。这种分层结构带来以下优势：

  • 资源共享：多个镜像可以共享相同的底层
  • 高效存储：只需保存变化的部分
  • 快速构建：利用缓存加速构建

Dockerfile 构建镜像

Dockerfile 是构建镜像的脚本文件，通过指令定义镜像的构建过程。

常用指令

指令 说明 示例
FROM 指定基础镜像 FROM python:3.8-slim
RUN 执行命令（构建时） RUN pip install flask
COPY 复制文件到镜像 COPY app.py /app/
ADD 高级复制（支持 URL 和解压） ADD file.tar.gz /app/
WORKDIR 设置工作目录 WORKDIR /app
ENV 设置环境变量 ENV APP_ENV=production
ARG 构建时变量 ARG VERSION=1.0
EXPOSE 声明容器端口 EXPOSE 8080
VOLUME 创建挂载点 VOLUME /data
CMD 容器启动命令（可被覆盖） CMD ["python", "app.py"]
ENTRYPOINT 容器启动命令（不可被覆盖） ENTRYPOINT ["python"]
LABEL 添加元数据 LABEL version="1.0"
USER 指定运行用户 USER appuser

Docker 网络

网络原理

安装 Docker 后，会创建一个名为 docker0 的虚拟网桥，默认 IP 网段为 172.17.0.0/16。

  • 每个容器启动时，会创建一对 veth pair 虚拟网卡
  • 一端连接容器（eth0），另一端连接 docker0 网桥
  • 容器通过 docker0 实现与宿主机及其他容器的通信

网络模式

模式 说明
bridge 默认模式，通过 docker0 网桥连接
host 容器直接使用宿主机网络栈
none 无网络，仅本地回环
container 共享另一个容器的网络栈

网络管理

自定义网络的优势
