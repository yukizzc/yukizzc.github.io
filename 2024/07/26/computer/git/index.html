项目级 项目目录\.git\config 对应 --local 参数，仅当前项目生效

核心概念与工作流程

Git 架构图

四个区域说明

区域 英文 说明
工作区 Workspace 本地项目目录，存放实际代码文件
暂存区 Index/Stage 临时存放修改，准备提交的文件列表
本地仓库 Repository 存储所有提交历史的地方（.git 目录）
远程仓库 Remote 托管代码的服务器（GitHub、Gitee 等）

Git 工作流程

文件状态

状态 说明 示例命令
未跟踪 新文件，从未提交过 git add 后变为已暂存
已暂存 已 add，等待提交 git commit 后变为未修改
已修改 文件已修改但未暂存 git add 变为已暂存
未修改 文件与上次提交一致 修改后变为已修改

Git GUI 使用

新建

  1. 创建本地仓库参考init命令

  2. 远程仓库通过网页创建同名仓库

    1. 不要勾选「初始化仓库」（保持空仓库）
    2. 创建后得到远程地址：https://gitee.com/用户名/xxxx.git

  3. 本地添加远程仓库地址

    1. 点击菜单栏 「Remote」 → 「Add…」

    2. 弹出窗口：

      字段 填写内容
      Name origin
      Location / URL https://gitee.com/用户名/xxxx.git

    3. 弹出窗口：点击 「Add」 确认

  4. 按下面基本操作把本地文件推送到远程

基本操作

提交本地修改到远程：

  1. Stage Changed（暂存修改）
  2. Commit（提交）
  3. Push（推送到远程）

从远程拉取更新：

  1. Remote → Fetch from → origin
  2. Merge → Local Merge → 选择 Tracking Branch
