系统目录结构

根目录及重要子目录

目录 说明 重要性
/ 根目录，Linux 文件系统的起点 ⭐⭐⭐⭐⭐
/bin Binary 缩写，存放常用命令（如 ls、mkdir、mv），所有用户都可执行 ⭐⭐⭐⭐⭐
/boot 存放 Linux 启动核心文件，包括内核和引导程序 ⭐⭐⭐⭐
/dev Device 缩写，存放外部设备文件（硬盘、光驱、鼠标等） ⭐⭐⭐⭐
/etc 重要：存放系统和程序的配置文件，类似 Windows 注册表 ⭐⭐⭐⭐⭐
/home 重要：普通用户的主目录，每个用户有自己命名的子目录 ⭐⭐⭐⭐⭐
/lib、/lib64 存放系统最基本的动态链接共享库（类似 Windows 的 DLL） ⭐⭐⭐⭐⭐
/media 系统自动挂载设备的目录（U盘、光驱等） ⭐⭐⭐
/mnt 用户临时挂载文件系统的目录（手动挂载额外硬盘） ⭐⭐⭐
/opt 重要：安装额外软件所用的目录，如 MySQL ⭐⭐⭐⭐
/proc 虚拟目录，系统内存映射，用于获取系统信息 ⭐⭐⭐⭐
/root 重要：超级管理员 root 用户的主目录 ⭐⭐⭐⭐⭐
/run 存放进程产生的临时文件，系统重启时清空 ⭐⭐⭐
/sbin Super User Bin，存放系统管理员使用的管理程序 ⭐⭐⭐⭐
/srv 存放服务启动后需要提取的数据 ⭐⭐⭐
/sys 内核设备树文件系统，反映内核设备信息 ⭐⭐⭐
/tmp 重要：存放临时文件，所有用户都可写 ⭐⭐⭐⭐
/usr 重要：存放应用程序和文件，类似 Windows 的 Program Files ⭐⭐⭐⭐⭐
/usr/bin 系统用户使用的应用程序 ⭐⭐⭐⭐⭐
/usr/sbin 超级用户使用的高级管理程序 ⭐⭐⭐⭐
/usr/src 内核源代码默认放置目录 ⭐⭐⭐
/usr/local 用户自行安装软件的目录 ⭐⭐⭐⭐⭐
/var 存放经常变化的文件，如各类日志文件 ⭐⭐⭐⭐⭐
/lost+found 系统非法关机后存放恢复文件，平时为空 ⭐⭐⭐

环境变量

LD_LIBRARY_PATH

LD_LIBRARY_PATH 是 Linux 环境变量，用于指定查找动态链接库（.so 文件）时的额外路径。

当程序需要动态链接库时，系统会按以下顺序查找：

  1. 编译时指定的 -rpath 路径
  2. LD_LIBRARY_PATH 环境变量指定的路径
  3. 系统默认路径（/lib、/usr/lib）
