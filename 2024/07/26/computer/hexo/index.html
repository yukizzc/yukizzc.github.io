Hexo 是什么

Hexo 是一个基于 Node.js 的静态博客生成器。你只需要用 Markdown 写文章，Hexo 会自动生成漂亮的 HTML 页面。

核心理念：关注内容，而非技术细节。

2. 安装必要依赖（部分主题需要）

3. 创建主题配置文件（可选但推荐）

把主题自带的配置文件复制一份到根目录，并改名。这样做的好处是：

  • 主题更新时，根目录的配置文件不会被覆盖
  • 所有自定义设置都在根目录这一个文件里，方便管理

4. 预览主题

外观修改

改外观、菜单、评论等，都直接在根目录的 _config.主题名.yml 中修改，保存后重新运行 hexo server 即可看到效果。

注意事项

  • 每个主题的配置项不同，具体功能请参考对应主题的官方文档
  • 如果预览时样式错乱，检查是否已安装主题所需的依赖包
  • 修改配置后需要重启 hexo server 才能生效

博客迁移

需要备份的文件

类型 文件/文件夹 说明
必须备份 source/ 所有文章和页面
必须备份 _config.yml Hexo 主配置
必须备份 _config.主题名.yml 主题自定义配置（如 _config.butterfly.yml）
必须备份 package.json 依赖列表
必须备份 package-lock.json 依赖版本锁定
可选备份 scaffolds/ 文章模板
可选备份 themes/ 如果主题是手动安装的（非 npm）
不需要备份 node_modules/ 依赖包，新电脑会自动安装
不需要备份 public/ 生成的静态文件
不需要备份 .cache/、db.json 临时文件

迁移步骤

1. 打包博客（排除 node_modules）

或手动复制除 node_modules、public 外的所有文件到新电脑。

2. 新电脑上解压并安装依赖

3. 预览验证

为什么不需要复制 node_modules？

  • node_modules 里的所有包（包括主题）都记录在 package.json 中
  • 在新电脑执行 npm install 会自动下载，且版本完全一致
  • 直接复制可能导致跨平台/Node 版本不兼容问题

执行 npm install 后，node_modules/hexo-theme-xxx 会自动下载。你的自定义配置在根目录 _config.主题名.yml 中，已备份。
