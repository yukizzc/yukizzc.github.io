Socket 基础

什么是 Socket

Socket（套接字）是 Python 标准库中的模块，用于实现网络通信。它屏蔽了底层网络协议的复杂性，为应用层提供统一的编程接口。

形象理解：Socket 就像是一个"电话机"，你需要知道对方的号码（IP）和分机号（端口），拨通后就可以通话（传输数据）。

核心概念

概念 说明 示例
套接字 (Socket) 网络通信的端点，结合 IP 地址和端口号 ('localhost', 8888)
协议 TCP（可靠连接）或 UDP（不可靠数据报） SOCK_STREAM / SOCK_DGRAM
IP 地址 标识网络设备 127.0.0.1（本地回环）、192.168.1.1
端口号 标识具体服务，范围 0-65535 80（HTTP）、443（HTTPS）、8888（自定义）

常用方法速查表

方法 说明 参数示例
socket.socket(family, type) 创建套接字 socket.AF_INET（IPv4）, socket.SOCK_STREAM（TCP）
socket.bind(address) 绑定地址 ('localhost', 8888)
socket.listen(backlog) 监听连接（服务端） 5（最大等待连接数）
socket.accept() 接受客户端连接（服务端） 返回 (client_socket, address)
socket.connect(address) 连接服务器（客户端） ('localhost', 8888)
socket.send(data) 发送数据（TCP） 需为 bytes 类型
socket.recv(bufsize) 接收数据（TCP） 1024（每次接收的最大字节数）
socket.sendto(data, address) 发送数据（UDP） 需指定目标地址
socket.recvfrom(bufsize) 接收数据（UDP） 返回 (data, address)
socket.close() 关闭套接字 无

UDP 编程

UDP（用户数据报协议）是无连接、不可靠的传输协议，数据包独立发送，不保证到达顺序和完整性，但速度快、开销小。

UDP 客户端

UDP 特点说明

  • 无需建立连接：客户端直接发送数据，服务端不需要 accept()
  • 每个数据包独立：sendto() 和 recvfrom() 成对使用
  • 可能丢包：不保证送达，适合实时性要求高的场景

TCP vs UDP 对比

特性 TCP UDP
连接方式 面向连接（三次握手） 无连接
可靠性 可靠（重传、排序、流量控制） 不可靠（可能丢包、乱序）
速度 较慢（因可靠性开销） 较快（无连接开销）
数据边界 流式传输，无边界 数据报传输，有边界
适用场景 文件传输、HTTP、邮件、数据库 视频流、DNS、实时游戏、VoIP
Python 方法 send() / recv() sendto() / recvfrom()

选择建议：

  • 需要数据完整可靠 → TCP
  • 追求速度、可容忍丢包 → UDP

WebSocket 编程

什么是 WebSocket

WebSocket 是一种应用层协议，运行在 TCP 之上。它通过 HTTP/HTTPS 建立初始连接（握手阶段），然后升级为持久化的双向通信通道。

为什么 WebSocket 基于 TCP？

  • TCP 提供可靠的连接和有序数据传输
  • 满足 WebSocket 对实时通信（聊天、通知推送）的可靠性需求

HTTP vs WebSocket 对比：

特性 HTTP WebSocket
通信方式 请求-响应（单向） 全双工（双向实时）
连接 短连接（每次请求新建） 长连接（一次握手持续通信）
适用场景 网页浏览、API 调用 聊天、股票行情、在线游戏
