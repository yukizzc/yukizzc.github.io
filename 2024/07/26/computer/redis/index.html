配置

redis.windows.conf文件包含了 Redis 在 Windows 上的配置设置，其中一些比较重要的设置包括

  1. bind：指定 Redis 监听的主机地址。
  2. port：指定 Redis 监听的端口号。
  3. requirepass：设置 Redis 访问密码，用于连接 Redis 服务器时进行身份验证。
  4. maxmemory：设置 Redis 实例能够使用的最大内存量，用于限制内存使用量。
  5. logfile：指定 Redis 日志文件的路径。
  6. dir：指定 Redis 工作目录，用于存储持久化文件和临时文件。
  7. save：设置自动持久化的规则RDB，指定在何种情况下进行数据持久化。
  8. appendonly：指定是否开启 AOF（Append Only File）持久化方式。
  9. appendfilename：设置 AOF 文件的名称。
  10. daemonize：指定是否以守护进程模式运行 Redis

默认# requirepass foobared，我们把注释去掉改成requirepass 1234就是设置好密码了

默认不开启AOF的，重启redis后优先AOF然后是RDB。RDB是每隔一段时间快照，AOF是完整的所有数据都会记录

python操作

创建redis对象

使用连接池是一种更好的做法，因为它提供了更好的性能和资源管理。因此，如果您的应用程序需要频繁地与Redis交互，或者在高并发环境中运行，那么使用连接池是明智的选择

decode_responses 参数的主要作用是控制从 Redis 返回的数据是否自动解码为字符串。它影响的是从 Redis 读取数据的解码行为，而不是数据存储时的编码方式。具体来说：

  • 当 decode_responses=False 时：
    • 从 Redis 获取的数据保持为字节字符串（bytes 类型）
    • 适用于存储和读取二进制数据的场景。如果你使用 pickle 序列化数据，读取数据时不能解码为字符串，必须使用 pickle.loads 进行反序列化
    • 直接读取的是一个字节字符串，此时后面跟上.decode(‘utf-8’)就能转换成字符串
  • 当 decode_responses=True 时：
    • 从 Redis 获取的数据会自动解码为字符串（使用指定的编码，默认是 UTF-8）
    • 适用于存储和读取字符串数据的场景。如果你存储的是 JSON 字符串，从 Redis 获取的数据会自动解码为 Python 字符串，这样你可以直接使用 json.loads 进行进一步处理

redis中存储都是字符、json、pickle这种，保存进int这种基础数据也是用字符存的，所以自己读取时候要注意要么全部序列化要么取回后从字符再转换基础格式

查看及删除

键值操作

  • set(key, value)：设置键值对
  • get(key)：获取指定键的值
  • delete(key)：删除指定键
  • exists(key)：检查指定键是否存在
  • expire(key, seconds)：设置键的过期时间（以秒为单位）
  • ttl(key)：获取键的剩余过期时间

只有键值这个有过期时间的参数

set(name, value, ex=None, px=None, nx=False, xx=False)

  • ex，过期时间（秒）
  • px，过期时间（毫秒）
  • nx，如果设置为True，则只有name不存在时，当前set操作才执行
  • xx，如果设置为True，则只有name存在时，当前set操作才执行

原子性

用incr取值时候能保持数据的原子性，它内部实现了自增1的操作，在多线程并发读取时候可以做到每次读取的数值都不一样

插入其他数据结构

用pickle进行编码和解码，这个相比json效率更好，而且支持序列化任意数据，缺点是他是二进制格式并且python专用

用json进行编码和解码，仅支持基本的数据类型（字符串、数字、列表、字典等），但是适合跨平台通用性好

列表操作

  • lpush(key, value)：在列表左侧插入一个或多个值
  • rpush(key, value)：在列表右侧插入一个或多个值
  • llen(key)：获取列表长度
  • lrange(key, start, end)：获取列表中指定范围内的值
  • lpop(key)：从列表左侧移除并返回一个值
  • rpop(key)：从列表右侧移除并返回一个值

哈希操作

哈希其实就是python的字典

  • hset(name, key, value)：设置哈希表中指定字段的值
  • hget(name, key)：获取哈希表中指定字段的值
  • hgetall(name)：获取哈希表中所有字段和值
  • hdel(name, key)：删除哈希表中指定字段
  • hexists(name, key)：检查哈希表中是否存在指定字段

集合操作

  • sadd(name, *values)：向集合添加一个或多个成员
  • smembers(name)：获取集合中的所有成员
  • sismember(name, value)：检查指定成员是否存在于集合中
  • srem(name, *values)：从集合中移除一个或多个成员

有序集合操作

  • zadd(name, mapping)：向有序集合添加一个或多个成员，同时指定分数。
  • zrange(name, start, end)：按照分数从低到高的顺序获取有序集合中指定范围的成员。
  • zrevrange(name, start, end)：按照分数从高到低的顺序获取有序集合中指定范围的成员。

aioredis

使用异步客户端（如 aioredis）的主要目的是提高客户端应用程序的性能，特别是在高并发场景下。异步客户端可以在等待 Redis 服务器响应时，不阻塞当前线程，从而可以在同一时间处理多个请求。这在 I/O 密集型应用中尤为重要，因为可以充分利用系统资源，提升吞吐量和响应速度

  • 在使用同步的 Redis 客户端时，每个命令都会阻塞当前线程
  • 在使用异步的 Redis 客户端时，命令是非阻塞的，可以同时发送多个命令而不需要等待前一个命令的完成
