报单

报单需要参数较多，现罗列如下。其中，赋值变量为外部传入的参数（如买卖方向、品种、价格、数量等）；其他参数大部分可直接写死（如投机套保标志固定为“投机”），如有特殊需求再行调整。

OrderRef 是本地会话全局唯一编号，必须保持递增且为数字。可由用户自行维护，也可由系统自动填写。本文采用 Redis 的原子递增生成。

CombOffsetFlag：开平标志

宏定义 值 含义
THOST_FTDC_OF_Open ‘0’ 开仓
THOST_FTDC_OF_Close ‘1’ 平仓
THOST_FTDC_OF_ForceClose ‘2’ 强平
THOST_FTDC_OF_CloseToday ‘3’ 平今
THOST_FTDC_OF_CloseYesterday ‘4’ 平昨
THOST_FTDC_OF_ForceOff ‘5’ 强减
THOST_FTDC_OF_LocalForceClose ‘6’ 本地强平

宏定义 值 含义
THOST_FTDC_D_Buy ‘0’ 买
THOST_FTDC_D_Sell ‘1’ 卖

宏定义 值 含义
THOST_FTDC_OPT_AnyPrice ‘1’ 任意价
THOST_FTDC_OPT_LimitPrice ‘2’ 限价
THOST_FTDC_OPT_BestPrice ‘3’ 最优价
THOST_FTDC_OPT_LastPrice ‘4’ 最新价
THOST_FTDC_OPT_LastPricePlusOneTicks ‘5’ 最新价浮动上浮1个tick
THOST_FTDC_OPT_LastPricePlusTwoTicks ‘6’ 最新价浮动上浮2个tick
THOST_FTDC_OPT_LastPricePlusThreeTicks ‘7’ 最新价浮动上浮3个tick
THOST_FTDC_OPT_AskPrice1 ‘8’ 卖一价
THOST_FTDC_OPT_AskPrice1PlusOneTicks ‘9’ 卖一价浮动上浮1个tick
THOST_FTDC_OPT_AskPrice1PlusTwoTicks ‘A’ 卖一价浮动上浮2个tick
THOST_FTDC_OPT_AskPrice1PlusThreeTicks ‘B’ 卖一价浮动上浮3个tick
THOST_FTDC_OPT_BidPrice1 ‘C’ 买一价
THOST_FTDC_OPT_BidPrice1PlusOneTicks ‘D’ 买一价浮动上浮1个tick
THOST_FTDC_OPT_BidPrice1PlusTwoTicks ‘E’ 买一价浮动上浮2个tick
THOST_FTDC_OPT_BidPrice1PlusThreeTicks ‘F’ 买一价浮动上浮3个tick
THOST_FTDC_OPT_FiveLevelPrice ‘G’ 五档价

ContingentCondition：触发条件

宏定义 值 说明
THOST_FTDC_CC_Immediately ‘1’ 立即生效（普通单）
THOST_FTDC_CC_Touch ‘2’ 止损单（需交易所支持）
THOST_FTDC_CC_TouchProfit ‘3’ 止盈单（需交易所支持）
THOST_FTDC_CC_ParkedOrder ‘4’ 预埋单（非交易时报入，开市后自动发出）
THOST_FTDC_CC_LastPriceGreaterThanStopPrice ‘5’ 最新价 > 条件价
THOST_FTDC_CC_LastPriceGreaterEqualStopPrice ‘6’ 最新价 ≥ 条件价
THOST_FTDC_CC_LastPriceLesserThanStopPrice ‘7’ 最新价 < 条件价
THOST_FTDC_CC_LastPriceLesserEqualStopPrice ‘8’ 最新价 ≤ 条件价
THOST_FTDC_CC_AskPriceGreaterThanStopPrice ‘9’ 卖一价 > 条件价
THOST_FTDC_CC_AskPriceGreaterEqualStopPrice ‘A’ 卖一价 ≥ 条件价
THOST_FTDC_CC_AskPriceLesserThanStopPrice ‘B’ 卖一价 < 条件价
THOST_FTDC_CC_AskPriceLesserEqualStopPrice ‘C’ 卖一价 ≤ 条件价
THOST_FTDC_CC_BidPriceGreaterThanStopPrice ‘D’ 买一价 > 条件价
THOST_FTDC_CC_BidPriceGreaterEqualStopPrice ‘E’ 买一价 ≥ 条件价
THOST_FTDC_CC_BidPriceLesserThanStopPrice ‘F’ 买一价 < 条件价
THOST_FTDC_CC_BidPriceLesserEqualStopPrice ‘H’ 买一价 ≤ 条件价

说明：

  • 当 ContingentCondition 为 '5' 及以上时，为条件单，需同时设置 StopPrice（条件价）
  • 条件单报入后存储在 CTP 服务端，待条件满足时自动提交交易所
  • StopPrice 仅在条件单场景下有效，普通单可忽略

TimeCondition、VolumeCondition、MinVolume 组合说明

字段 普通单 FAK FOK
TimeCondition '3'（当日有效） '1'（IOC） '1'（IOC）
VolumeCondition '1'（任何数量） '1'（任何数量）或 '2'（最小数量） '3'（全部数量）
MinVolume 无需填写 当 VolumeCondition = '2' 时，填写最小成交手数 无需填写

各类型说明

类型 说明
普通单 当日有效，按指定价格成交，未成交部分保留在盘中
FAK（Fill and Kill） 立即成交剩余撤销，未成交部分立即撤销
FOK（Fill or Kill） 立即全部成交否则撤销，必须整单全部成交，否则整单撤销

报单回报

报单回报的来回图见图片

常用字段说明

OrderStatus：报单状态

报单状态流转如下：

  • 正常流程：'a'（未知/初始） → '3'（未成交在队列） → '0'（全部成交）或 '1'（部分成交）
  • 撤单流程：'a'（未知/初始） → '3'（未成交在队列） → '5'（已撤单）
  • 部分成交后撤单：'1'（部分成交） → '5'（已撤单）

重要说明：

  • 状态 '3'（未成交在队列）和 '5'（已撤单）不会改变实际持仓
  • 状态 '1'（部分成交）会部分改变实际持仓（已成交部分生效）
  • 状态 '0'（全部成交）完全改变实际持仓
  • 实际持仓的变化取决于成交（Trade），而非报单状态本身。只要有成交回报（OnRtnTrade），持仓就会变化，即使报单状态还是 '1'（部分成交）

枚举定义

宏定义 值 含义
THOST_FTDC_OST_AllTraded ‘0’ 全部成交
THOST_FTDC_OST_PartTradedQueueing ‘1’ 部分成交，剩余在队列
THOST_FTDC_OST_PartTradedNotQueueing ‘2’ 部分成交，剩余不在队列
THOST_FTDC_OST_NoTradeQueueing ‘3’ 未成交，仍在队列
THOST_FTDC_OST_NoTradeNotQueueing ‘4’ 未成交，不在队列
THOST_FTDC_OST_Canceled ‘5’ 已撤单
THOST_FTDC_OST_Unknown ‘a’ 未知（初始状态）
THOST_FTDC_OST_NotTouched ‘b’ 尚未触发（条件单）
THOST_FTDC_OST_Touched ‘c’ 已触发（条件单）

CombOffsetFlag：开平标志

宏定义 值 含义
THOST_FTDC_OF_Open ‘0’ 开仓
THOST_FTDC_OF_Close ‘1’ 平仓
THOST_FTDC_OF_ForceClose ‘2’ 强平
THOST_FTDC_OF_CloseToday ‘3’ 平今
THOST_FTDC_OF_CloseYesterday ‘4’ 平昨
THOST_FTDC_OF_ForceOff ‘5’ 强减

CombHedgeFlag：投机套保标志

宏定义 值 含义

THOST_FTDC_HF_Arbitrage ‘2’ 套利
THOST_FTDC_HF_Hedge ‘3’ 套保
THOST_FTDC_HF_MarketMaker ‘5’ 做市商
THOST_FTDC_HF_SpecHedge ‘6’ 第一腿投机，第二腿套保
THOST_FTDC_HF_HedgeSpec ‘7’ 第一腿套保，第二腿投机

OrderRef vs OrderSysID

字段 说明
OrderRef 报单引用（本地编号），下单时由客户端生成，需保持当天唯一且为数字。用于本地追踪订单
OrderSysID 报单编号（柜台编号），报单成功发送至柜台后由柜台返回。有此字段说明报单已正确送达柜台

建议：撤单时使用 OrderSysID 进行撤单操作，通过将 OrderRef 与 OrderSysID 绑定来管理订单生命周期。

成交回报

常用字段说明

基础信息

字段 说明
TradeTime 成交时间，格式为 HH:MM:SS
ExchangeID 交易所代码，如 'CFFEX'（中金所）、'SHFE'（上期所）、'DCE'（大商所）、'CZCE'（郑商所）
InstrumentID 合约代码，如 'rb2505'
TradeID 成交编号，交易所生成的唯一成交标识

成交信息

字段 说明
Price 成交价格
Volume 本次成交数量（手）
Direction 买卖方向：'0'（买）、'1'（卖）
OffsetFlag 开平标志：'0'（开仓）、'1'（平仓）、'3'（平今）、'4'（平昨）
TradeType 成交类型，通常为普通成交

关联标识

字段 说明 用途
OrderRef 报单引用（本地编号） 关联本地报单记录
OrderSysID 报单编号（柜台编号） 关联柜台报单记录

注意：一个报单可能对应多笔成交（如分批成交），通过 OrderRef 或 OrderSysID 可关联到对应的报单，通过 TradeID 区分同一报单下的不同成交记录。

区别

说明：

  • 一个报单可能对应多笔成交（如大单分批成交）
  • 通过 OrderRef / OrderSysID 将成交与报单关联
  • 通过 TradeID 区分同一报单下的不同成交
  • 累加同一报单的所有成交 Volume 可得到该报单的总成交数量
